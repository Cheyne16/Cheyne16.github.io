2-聚类分析

1 min · 156 words · Chan | Suggest Changes

有监督学习：分类、回归、目标检测、语义切分、图像描述。

无监督学习：学习关于数据的某种隐藏结构。聚类、特征降维、特征学习、密度估计。

聚类#

类簇划分的角度：

  • 良态可分类
  • 基于最近邻的聚类
  • 基于密度的聚类
  • 基于中心点的聚类
  • 基于共有属性或概念的聚类

聚类方法：

  • 基于划分（Partitional Clustering）
  • 基于层次（Hierarchical clustering）
    • 自顶向下
    • 自底向上
  • 其它：
    • 排它性聚类 vs. 非排它性聚类
    • 模糊聚类 vs. 非模糊聚类
    • 部分聚类 vs. 完全聚类
    • 异构型聚类 vs. 同构型聚类

聚类模型#

K-means#

基于中心点

思想：类内距离最小化，类间距离最大化

算法过程：

  • 假设有k个类簇，每个簇一个中心点(centriod)【中心点可看作有监督学习中的标签y】

  • 将每个样本点划分到距离最近的中心点所属的簇中

  • 迭代：

    • 更新每个簇的中心点
    • 然后重新划分

  • 直到中心点不变

转换为有监督学习模型#

目标函数：每个样本点与其簇中心点的距离之和。这里的距离可以是各种距离。

  • μk 表示簇Ck 的中心点（或其它能代表Ck的点）
  • 若x~n~被划分到簇C~k~则r~nk~=1，否则r~nk~= 0

优化目标：找到簇的中心点μk及簇的划分r~nk~使得目标函数SSE最小

算法过程：

  • choose some initial values for the μ~k~ (k=1,…,K)

  • 迭代：

    • minimize SSE with respect to the r~nk~ (n=1,…,N)

    • minimize SSE with respect to the m~k~

  • 直到收敛

注意#

  • k通常是人类专家确定，如何更好的确定k：

    • 可以借助层次聚类，比如在自底向上合并时，当距离比较大时，就可以作为不同的簇类。
    • k越大，sse越小，k=n时，sse=0
    • Gap统计量
    • 交叉验证
    • 重采样法
    • 其它非参方法

  • 初始中心点通常是随机选取，如何更好地选择初始中心点：

  • 算法复杂度：O( n × d × K × I )。n=样本点个数，d = 样本特征维度，K = 类的个数，I = 迭代次数

  • 中心点通常是当前簇中所有样本点的均值（K-medoids：中位数）

缺点#

类簇具有以下特征时，K-means表现不好，因为它得到的结果总是趋向于分布均匀且边界为球形：

  • 大小不同
  • 密度不同
  • 有非球形的分布形状

如何将解决这个问题：高斯混合模型

HAC#

基于层次

层次凝聚聚类（Hierarchical Agglomerative Clustering (HAC）：一种自底向上的聚类

（自顶向下（divisive））

基于层次不会得到扁平的聚类，而是嵌套的、层次化的类别树，根据此树即可得到想要的聚类结果。如下图，可以随意划分得到4个类簇、3个或2个。

算法思想：

  • 初始：每个点都是一个类簇，建立n×n的相似度矩阵
  • 不断合并最近的两个类簇，更新n’×n’相似度矩阵
  • 直到只有一个类簇

如何衡量两个类簇的距离（相似度）呢？有多种方法，按照情况选用

  • MIN (single linkage)
  • MAX (complete linkage)
  • Group Average
  • Distance Between Centroids
  • Other methods driven by an objective function
    • Ward’s Method uses squared error

高斯混合模型（及EM算法#

上述方法使用算法划分类簇（硬分割），还有另一类方法是使用模型，将数据的概率分布描述为一个混合分布模型（这样就可以软分割，求得样本点在每个类簇上的概率），常用高斯混合模型。即多个高斯分布的线性叠加，如 G(x) = w~1~G~1~ + w~2~G~2~ + w~3~G~3~ + …

那么问题来了

  • 由这个模型如何得到x的划分？

  • 如何估计模型参数（G中的参数）？（EM算法）
