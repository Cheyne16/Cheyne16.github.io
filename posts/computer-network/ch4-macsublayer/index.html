4.2.2 CSMA#

Carrier Sense Multiple Access Protocols

  • CS：载波监听。检测电压摆动值。

  • MA：多点接入。

局域网中：==传播时间远远小于发送时间==，因为是在局域网内，距离短

协议思想：

  • 发送之前，先监听信道（避免）

    • 信道空闲：以一定概率发送帧

    • 信道忙：等待直到信道空闲

      • 非坚持CSMA：等待一个随机时间再重新监听载波

      • 坚持CSMA：一直监听着

        • 1-坚持CSMA：一直监听信道，空闲了直接发送
        • p-坚持CSMA：一直监听信道，空闲了以概率p发送，以概率p-1延迟一个随机时间后再发送

  • 如果冲突：等待一个合理时间（RTT+ACK竞争时间）

  • 如果未收到ACK，则重传

CSMA比较：

1-坚持非坚持
低负载高吞吐量、低延迟
高负载低吞吐量高吞吐量

评价MAC协议的指标： （低负载适合使用竞争方法）

  • 低负载（轻载）情况下的时延

  • 高负载（重载）情况下的吞吐量（信道利用率）

冲突时间：τ（propagation delay）

CSMA/CD ==以太网== IEEE 802.3#

半双工通信系统；节省时间和宽带；以太网的技术基础

基本思想：

  • 载波监听（避免冲突）
  • 冲突检测（处理冲突）——边发边检测冲突，检测到冲突后：停止发送 + 发送jam信号
    • 冲突检测原理：发的同时往回收，检测信号是否相同
      • 无线网络发出的信号是接受信号的100万倍强度，覆盖了，收不回；
      • 还与调制模式有关
  • 退避（处理冲突）——等待随机时间后重传

状态：CSMA/CD 可能处于以下三个状态之一: 竞争，传输，空闲。

冲突\竞争时间：2τ

1-persistent CSMA/CD的二进制指数退避算法：。。。重发次数越多，等待时间越长

性能: 帧时远大于传播时间时，信道效率高

4.2.5 无线局域网协议#

无线传输需要解决的问题：

  • 传输范围有限
  • 并非所有站都在彼此范围内，所以可能检测不出碰撞
  • 干扰出现在接收端，而不是发送端
  • CSMA/CD不适用（隐藏/暴露终端问题；大多数无线电设备都是半双工的它们不能同时在一个频率上既传输数据，又监听噪声尖峰；无线信号发出的信号是接受信号的100万倍强度，覆盖了，收不回；）

隐藏终端和暴露终端： 都是因为CSMA/CD协议的LBT（Listen Before Talk）机制导致的。

  • 隐藏终端：【A B C】A给B发，C没有监听到A的信号，也给B发，导致B处冲突。 无线通信都要考虑隐藏终端。
  • 暴露终端：【A B C D】A给B发，C监听到A的信号，误以为他现在不能给D发。 只会造成效率低。

无线网络为解决上述终端问题使用的办法：站通过握手来确定谁发送

  • MACA: 冲突避免多路访问——解决了上面俩问题
  • CSMA/CA: 具有冲突避免的载波监听多路访问（IEEE802.11）(wifi使用的)——只解决了隐藏终端问题

IEEE 802.3 —— 1-坚持CSMA/CD

广播式传输，所有站点都可以发送/收到数据，通过地址判断是否接收。

帧时间隔（Inter Frame Gap）： 用于模式转换，在IEEE 802.3中时间为传输96bit（更短可以是47bit）的时间，对于10M bps Ethernet就是9.6μs。监听到信道空闲时间超过IFG时才发送帧。

工作流程：

  • 监听是否空闲
  • 空闲时间超过IFG，发送帧
  • 信道忙，持续监听
  • 发送时检测冲突（2tao时间）
  • 在2tao时间内：有冲突，停止发送，发送jam，退避；无冲突：ok

习题知识点#

  • VLAN只需要交换机或网桥支持就可以，如果用户设备不支持，则交换机在转发给用户设备时把帧中VLAN相关的tag域去掉就可以了

  • 典型的网络地址IP地址就是一个IP对应一个网络接口，路由器至少具有两个或两个以上的IP地址

    多个地址是可以对应一个网络接口的，比如一个接口既有一个单播地址，也可以同时属于多个组，那么就对应多个组播地址，所以一个网络接口可以具有多个网络地址，但是一个网络地址不能同时属于多个网络接口
