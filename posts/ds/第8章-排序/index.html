第8章 排序

November 15, 2022 · 2 min · 378 words · Chan | Suggest Changes

概念#

排序算法的稳定性：排序之后元素的相对顺序不变。

稳定的是否一定比不稳定的好：不一定，看实际需求。

分类：

  • 内部排序：数据都在内存中——关注时间、空间复杂度
  • 外部排序：数据太多，无法全部放入内存——还要关注如何使磁盘读写次数更少

插入排序#

直接插入排序#

普通实现：

void InsertSort(int A[], int n) {
	int i, j, temp;
	for (i=1; i<n; i++)
		if (A[i]<A[i-1]) {
			temp = A[i];
			for (j=i-1; j>=0 && A[j]>temp; --j)
				A[j+1] = A[j];
			A[j+1] = temp;
		}
}

带哨兵实现：不用每轮都判断 j>=0

void InsertSort(int A[], int n) {
	int i, j;
	for (i=2; i<n; i++)
		if (A[i]<A[i-1]) {
			A[0] = A[i];
			for (j=i-1; A[j]>A[0]; --j)
				A[j+1] = A[j];
			A[j+1] = A[0];
		}
}

算法效率分析：

  • ==稳定==

  • 空间复杂度：==O(1)==

  • 时间复杂度：比较+移动

    • 最好情况：已经是顺序的，只需比较n-1次，故 ==O(n)==
    • 最坏情况：逆序，需要n-1趟，每趟对比关键字2到n次，移动3到n+1次，故 ==O(n^2^)==
    • 平均：==O(n^2^)==

  • 适用于==顺序表、链表==，链表时：移动次数变少，但是比较次数不变，所以时间复杂度仍是==O(n^2^)==

优化：折半插入排序#

注意：何时停止查找，当前元素的插入位置

算法复杂度：比较的次数减少，但是移动次数没变，所以时间复杂度仍是==O(n^2^)==

它不适合链式。

希尔排序#

又叫缩小增量排序。先部分有序，再全局有序。

上述代码中由于每个步长中执行 i++，所以并不是处理完一个子表再处理另一个，而是交替处理。

复杂度分析：

  • 稳定性：==不稳定==，子表内的交换可能会打乱原有顺序。

  • 空间复杂度：==O(1)==

  • 时间复杂度：无法用数学手段证明确切的时间复杂度

    • 最坏情况：d=1，变成直接插入排序，==O(n^2^)==
    • 当n在某个范围内时，可达 ==O(n^1.3^)==

  • 仅适用于==顺序表==，因为处理子表需要随机访问。

冒泡排序#

基于交换的排序：冒泡、选择。

算法性能分析：

  • ==稳定==：相等时不交换
  • 可==链表==：从头到尾冒

快速排序#

基于交换的排序：冒泡、选择。

一趟排序（或者说一次划分）确定==一个元素的最终位置==，这个元素称作枢轴。然后对划分得到的子表们重复上述划分过程，直至子表内只有一个元素或空为止，则所有元素都放在了最终位置上。

枢轴元素的选取一般选第一个元素或更好的方式：

  • 取头、中、尾的中间值
  • 随便选

算法步骤：

  1. 从数列中挑出一个元素，称为 “基准”（pivot）;
  2. 重新排序数列，所有元素比基准值小的摆放在基准前面，所有元素比基准值大的摆在基准的后面（相同的数可以到任一边）。在这个分区退出之后，该基准就处于数列的中间位置。这个称为分区（partition）操作；
  3. 递归地（recursive）把小于基准值元素的子数列和大于基准值元素的子数列排序；
void QuickSort(int A[], int low, int high) {
	if (low<high) {  // 递归跳出的条件
        int pivotpos = Partition(A, low, high); // 划分
        QuickSort(A, low, pivotpos-1);          // 划分左子表
        QuickSort(A, pivotpos+1, high);         // 划分右子表
    }
}
int Partition(int A[], int low, int high) {
    int pivot = A[low];      // 第一个元素作为枢轴
    while (low < high) {     // 用low、high搜索枢轴的最终位置
        while (low<high && A[high]>=pivot) --high;
        A[low] = A[high];    // 比枢轴小的元素移动到左端
        while (low<high && A[low]<=pivot) ++low;
        A[high] = A[low];    // 比枢轴大的元素移动到右端
    }
    A[low] = pivot;          // 枢轴元素存放到最终位置
    return low;              // 返回存放枢轴的最终位置
}

算法性能分析：

  • ==不稳定==，如 2 2 1。

    快排和冒泡都是基于交换的排序，但是冒泡的交换是相邻两个元素的交换，所以不会乱序，但是快排是不相邻的交换，所以可能会乱序。

    )：既然那么快，就要有缺点

  • 空间复杂度：==O(递归层数)==

  • 时间复杂度：每一层的quicksort函数都只需要处理剩余的待排序元素，时间复杂度不超过O(n)，所以总的时间复杂度为==O(n*递归层数)==

把n个元素组织成二叉树，二叉树的层数就是递归调用的层数。n个结点的二叉树的最小高度为 ⌊log~2~n⌋+1，最大高度为 n。

快排优化思路：

  • 选头、中、尾三个位置的元素，取中间值作为枢轴元素
  • 随机选择一个作为枢轴元素

简单选择排序#

选择排序：简单选择排序、堆排序

算法效率分析：

  • ==不稳定==，如 2 2 1。

    快排和冒泡都是基于交换的排序，但是冒泡的交换是相邻两个元素的交换，所以不会乱序，但是快排是不相邻的交换，所以可能会乱序。

    简单选择排序同快排，都会不相邻交换，因此不稳定。

  • 空间复杂度：==O(1)==

  • 时间复杂度：无论什么情况都需要选择n-1趟，故 ==O(n^2^)==

  • 适用于==顺序表、链表==

堆排序#

选择排序：简单选择排序、堆排序

  • 大根堆：根 ≥ 左、右，用于构造升序序列
  • 小根堆：根 ≤ 左、右，用于构造降序序列

排序过程：（大根堆为例）

  • 对给定的序列建立堆
    • 把所有==非终端结点==（≤⌊n/2⌋）检查一遍，如果不符合堆的要求则交换它和孩子结点（若建立大根堆，则与更大的孩子互换）
    • 元素下坠可能导致下一层的子树不符合要求，此时需要采用相同的方法==继续向下调整==（平衡二叉树删除操作时，不平衡会向上传导）
  • 每一趟从==堆顶==取出最大（小）的元素和待排序序列中==最后一个元素==交换位置，并将==待排序序列==再次调整为大根堆（堆顶下坠）
  • 经过n-1趟即可得到有序序列

代码：

// 建立大根堆
void BuildMaxHeap(int A[], int len) {
    for (int i=len/2; i>0; i--)   // 从后往前调整所有非终端结点
        HeadAdjust(A, i, len);
}

// 将以k为根的子树调整为大根堆
void HeadAdjust(int A[], int k, int len) {
    A[0]=A[k]; // 暂存子树的根节点
    for (int i=2*k; i<len; i*2) { // 沿key较大的子节点向下筛选
        if (i<len && A[i]<A[i+1])
            i++;                  // 取key较大的子节点的下标
        if (A[0]>=A[i]) break;    // 筛选结束
        else {
            A[k]=A[i];            // 将A[j]调整到双亲结点上
            k=i;                  // 修改k值，以便继续向下筛选
        }
    }
    A[k]=A[0];
}

// 堆排序的完整逻辑
void HeapSort(int A[], int len) {
    BuildMaxHeap(A, len);          // 初始建堆
    for (int i=len; i>1; i--) {    // n-1趟的交换和调整过程
        swap(A[i], A[1]);          // 堆顶元素和待排序序列中最后一个元素交换
        HeadAdjust(A, 1, i-1);     // 剩余待排序序列调整为堆
    }
}

算法效率分析：

  • ==不稳定==，如 1 2 2。

    快排和冒泡都是基于交换的排序，但是冒泡的交换是相邻两个元素的交换，所以不会乱序，但是快排是不相邻的交换，所以可能会乱序。

    堆排序中第二步，堆顶和堆底元素互换，是不相邻交换，因此不稳定。同快排。

  • 空间复杂度：==O(1)==

  • 时间复杂度：==O(nlog~2~n)==

    • 建堆：==O(n)==

    • 排序：==O(nlog~2~n)==

总结#

复杂度分析#

空间复杂度：

  • 快排：递归栈，却决于二叉树的高度，最好O(log~2~n)，最坏O(n)

  • 归并：借用了长度为n的辅助空间，故O(n)，有递归栈但是不如辅助空间大

  • 基数：O(r)

  • 插入、冒泡、选择：只是比较和交换，故O(1)

时间复杂度：

  • 基于比较的排序算法时间复杂度最低为O(nlog~2~n)，只有：快排，堆排，归并
  • 其它 O(n^2^)
